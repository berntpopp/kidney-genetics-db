## Issue #124: `$` in passwords breaks Docker Compose

### Assessment

Docker Compose interpolates `$VAR` and `${VAR}` in `.env` files. Passwords with `$` silently corrupt. This is a **documentation fix** — the code is correct, Docker Compose behavior is the issue.

### Decision: Documentation + safe password generation guidance

Per Docker Compose docs: `$$` escapes to literal `$`, single quotes prevent interpolation. Best practice: generate passwords without special characters (`openssl rand -hex 32`).

## Issue #125: Admin password not updated on restart

### Assessment

`create_default_admin()` in `database_init.py` only creates the admin if missing. When admin exists, it updates role/active flags but **never the password**. This means changing `ADMIN_PASSWORD` in `.env` has no effect on existing deployments.

### Decision: Add `ADMIN_FORCE_PASSWORD_RESET` env var

Per best practices (Django, Rails, Harbor all use create-only), we should NOT auto-sync the password on every restart. Instead, add an opt-in `ADMIN_FORCE_PASSWORD_RESET=true` env var that triggers a one-time password update. This covers the "locked out" recovery scenario without the security risk of always syncing.

Also document that `ADMIN_PASSWORD` is only used for initial creation.

## Issue #127: annotation_sources table not seeded on fresh deployment

### Assessment

`annotation_sources` records are created lazily by `BaseAnnotationSource._get_or_create_source()` when a source is first instantiated. But `_get_sources_to_update()` queries the table BEFORE instantiating sources — chicken-and-egg. On fresh deployment, no sources exist, so the pipeline exits with "nothing to do."

### Decision: Application-level startup seeding

Per best practices (and our own Alembic migration gotcha), we should NOT use Alembic data migrations. Instead, add a `seed_annotation_sources()` function to `database_init.py` that runs on every startup (idempotent, like `register_data_sources()`). This follows the existing pattern in `startup.py`.

```python
async def seed_annotation_sources(db: Session) -> int:
    """
    Seed annotation_sources table with all known sources.

    Idempotent: creates only sources that don't already exist.
    Called on every startup to ensure fresh deployments have all sources.

    Returns:
        Number of newly created sources.
    """
    from app.models.gene_annotation import AnnotationSource
    from app.pipeline.annotation_pipeline import AnnotationPipeline

    pipeline = AnnotationPipeline(db)
    created = 0

    for source_name, source_cls in pipeline.annotation_sources.items():
        existing = (
            db.query(AnnotationSource)
            .filter_by(source_name=source_name)
            .first()
        )
        if not existing:
            source_record = AnnotationSource(
                source_name=source_name,
                display_name=getattr(source_cls, "display_name", source_name),
                version=getattr(source_cls, "version", "1.0"),
                is_active=True,
                config={},
            )
            db.add(source_record)
            created += 1

    if created:
        db.commit()
        await logger.info(
            "Seeded annotation sources",
            created=created,
            total=len(pipeline.annotation_sources),
        )

    return created
```

## Issue #129: Global per-domain rate limiting

### Assessment

Current rate limiting: each annotation source calls `await self.apply_rate_limit()` which does a simple `asyncio.sleep(1/rate)`. Problems:
- Per-source, not per-domain — sources hitting the same domain don't coordinate
- No concurrency control — no semaphore limiting parallel requests
- 429 handling is per-request, not queue-level

### Decision: httpx-limiter at transport layer

After research, `httpx-limiter` (built on `aiolimiter`) provides exactly what's needed: a custom `AsyncBaseTransport` wrapper with per-domain rate limits. This is transparent to all existing code — no source changes needed except removing manual `apply_rate_limit()` calls.

However, `httpx-limiter` is a relatively new/small library. A **simpler approach** is to build a lightweight `DomainRateLimiter` using `aiolimiter` directly, integrated into the existing `RetryableHTTPClient`. This gives us full control and avoids a new dependency that may not be well maintained.

```python
"""
Per-domain rate limiter for outgoing HTTP requests.

Uses aiolimiter (leaky bucket) + asyncio.Semaphore for per-domain
rate limiting and concurrency control. Designed for annotation
pipeline sources that hit multiple external APIs.
"""

import asyncio
import time
from contextlib import asynccontextmanager
from typing import Any
from urllib.parse import urlparse

from aiolimiter import AsyncLimiter

from app.core.logging import get_logger

logger = get_logger(__name__)


class DomainLimiter:
    """Rate limiter + concurrency control for a single domain."""

    def __init__(self, max_rate: float = 5.0, max_concurrent: int = 3):
        self.max_rate = max_rate
        self._rate_limiter = AsyncLimiter(max_rate=max_rate, time_period=1.0)
        self._semaphore = asyncio.Semaphore(max_concurrent)
        self._paused_until: float = 0.0

    @property
    def is_paused(self) -> bool:
        return time.monotonic() < self._paused_until

    def pause(self, seconds: float) -> None:
        """Pause all requests to this domain (e.g., on 429 Retry-After)."""
        self._paused_until = time.monotonic() + seconds
        logger.sync_info(
            "Domain paused",
            pause_seconds=seconds,
        )

    @asynccontextmanager
    async def acquire(self):
        """Acquire rate limit slot + concurrency semaphore."""
        # Wait for pause to expire
        if self.is_paused:
            wait_time = self._paused_until - time.monotonic()
            if wait_time > 0:
                await asyncio.sleep(wait_time)

        async with self._semaphore:
            await self._rate_limiter.acquire()
            yield


class DomainRateLimiterRegistry:
    """
    Registry of per-domain rate limiters.

    Usage:
        registry = DomainRateLimiterRegistry(
            domain_configs={
                "rest.ensembl.org": {"max_rate": 15.0, "max_concurrent": 5},
                "eutils.ncbi.nlm.nih.gov": {"max_rate": 10.0, "max_concurrent": 3},
            },
        )
        limiter = registry.get("rest.ensembl.org")
        async with limiter.acquire():
            response = await client.get(url)
    """

    def __init__(
        self,
        domain_configs: dict[str, dict[str, Any]] | None = None,
        default_max_rate: float = 5.0,
        default_max_concurrent: int = 3,
    ):
        self._configs = domain_configs or {}
        self._default_max_rate = default_max_rate
        self._default_max_concurrent = default_max_concurrent
        self._limiters: dict[str, DomainLimiter] = {}
        self._lock = asyncio.Lock()

    def get(self, domain: str) -> DomainLimiter:
        """Get or create a rate limiter for a domain."""
        if domain not in self._limiters:
            config = self._configs.get(domain, {})
            self._limiters[domain] = DomainLimiter(
                max_rate=config.get("max_rate", self._default_max_rate),
                max_concurrent=config.get("max_concurrent", self._default_max_concurrent),
            )
        return self._limiters[domain]

    def get_for_url(self, url: str) -> DomainLimiter:
        """Extract domain from URL and return its limiter."""
        domain = urlparse(url).hostname or "unknown"
        return self.get(domain)

    def pause_domain(self, domain: str, seconds: float) -> None:
        """Pause all requests to a domain (call on 429 response)."""
        self.get(domain).pause(seconds)
```

```python
    async def apply_rate_limit(self, url: str | None = None) -> None:
        """Apply per-domain rate limiting before a request."""
        from app.core.domain_rate_limiter import get_domain_rate_limiter_registry

        if url:
            registry = get_domain_rate_limiter_registry()
            limiter = registry.get_for_url(url)
            async with limiter.acquire():
                return
        else:
            # Fallback to simple delay for sources without URL context
            delay = 1.0 / self.requests_per_second
            await asyncio.sleep(delay)
```

> **Note:** This is backward-compatible — sources that call `apply_rate_limit()` without a URL still get the old behavior. Sources that pass a URL get per-domain rate limiting. Migration to URL-based calls can be done incrementally.
